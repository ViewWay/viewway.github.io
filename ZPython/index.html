Architecture Overview

ZPython follows a classic compiler-and-VM pipeline. Python source code is parsed into an AST by a PEG parser, compiled to bytecode, and then executed by a stack-based virtual machine with a multi-strategy garbage collector.

Garbage Collector Design

ZPython employs a hybrid, multi-strategy garbage collector that combines several complementary techniques to achieve low-latency, high-throughput memory management:

  • Reference Counting — Primary reclamation mechanism. Every object carries a refcount that is incremented/decremented on assignment. Zero-count objects are freed immediately, providing deterministic deallocation and low pause times.
  • Mark-and-Sweep — Handles reference cycles that refcounting alone cannot collect. Periodic full-heap scans mark reachable objects from roots and sweep unreachable ones.
  • Generational Collection — Objects are divided into generations based on survival time. Young generations are collected frequently (most objects die young), while old generations are collected less often, reducing overall GC overhead.
  • Concurrent Collection — GC work is offloaded to a background thread where possible, minimizing stop-the-world pauses and keeping mutator throughput high.
  • Hybrid Orchestration — A policy layer dynamically selects the most appropriate GC strategy based on heap pressure, allocation rate, and fragmentation, blending the above strategies for optimal performance.

Quick Start

Prerequisites

Zig 0.13+ (or a recent nightly build). Clone and build:

git clone https://github.com/ViewWay/ZPython.git cd ZPython zig build -Doptimize=ReleaseFast

Run a Python File

./zig-out/bin/zpython hello.py

Interactive REPL

./zig-out/bin/zpython >>> def fib(n): ... if n <= 1: ... return n ... return fib(n - 1) + fib(n - 2) ... >>> fib(20) 6765

Example: hello.py

# hello.py — ZPython demo class Greeter: def __init__(self, name): self.name = name def greet(self): return f"Hello, {self.name}!" g = Greeter("ZPython") print(g.greet()) # Hello, ZPython!
