Architecture Overview

Nexus uses a thread-per-core model backed by Linux io_uring for zero-overhead async I/O. Each worker thread owns a dedicated event loop — no cross-thread lock contention, no work-stealing overhead.

Request Flow (Linux io_uring) ┌──────────────────────────────────────────────────────────┐ │ TCP Listener │ │ (SO_REUSEPORT, per-thread) │ └──────────────┬───────────────────────────────┬───────────┘ │ │ ┌─────────▼─────────┐ ┌─────────▼─────────┐ │ Worker Thread 0 │ │ Worker Thread N │ │ ┌───────────────┐ │ │ ┌───────────────┐ │ │ │ io_uring SQ │ │ ... │ │ io_uring SQ │ │ │ │ io_uring CQ │ │ │ │ io_uring CQ │ │ │ └───────┬───────┘ │ │ └───────┬───────┘ │ │ │ │ │ │ │ │ ┌───────▼───────┐ │ │ ┌───────▼───────┐ │ │ │ HTTP Parser │ │ │ │ HTTP Parser │ │ │ │ Router │ │ │ │ Router │ │ │ │ Middleware │ │ │ │ Middleware │ │ │ │ Handler │ │ │ │ Handler │ │ │ └───────────────┘ │ │ └───────────────┘ │ └────────────────────┘ └────────────────────┘

Key design decisions:

  • Thread-per-core — each CPU core runs one worker; zero lock contention on the hot path
  • io_uring submission queues — batched async syscalls without kernel transitions
  • SO_REUSEPORT — kernel-level load balancing across worker threads
  • Ownership model — each thread owns its connections end-to-end (accept, read, parse, route, respond)
  • Cross-platform fallback — epoll (Linux), kqueue (macOS/BSD) backends for non-io_uring environments

Quick Start

Add Nexus to your Cargo.toml:

[dependencies] nexus = { version = "0.1", features = ["full"] }

Write your first server:

use nexus::prelude::*; fn main() -> Result<()> { let app = App::new() .route("/", get("Hello from Nexus!")) .route("/api/health", get(health_handler)) .middleware(cors::default()) .middleware(logger::new()); Server::builder() .bind("0.0.0.0:8080")? .workers(num_cpus::get()) .run(app) } async fn health_handler() -> Json<HealthResponse> { Json(HealthResponse { status: "ok".into() }) } // With circuit breaker + retry for downstream calls: use nexus::resilience::{CircuitBreaker, Retry}; async fn call_service() -> Result<Response> { let cb = CircuitBreaker::builder() .failure_threshold(5) .reset_timeout(Duration::from_secs(30)) .build(); let retry = Retry::exponential_backoff(3); let client = HttpClient::new() .with(cb) .with(retry); client.get("https://api.example.com/data").await }

Feature Highlights

High Availability Patterns

Circuit breaker with configurable thresholds, exponential backoff retry, bulkhead isolation, graceful degradation with fallbacks. Production-tested resilience stack.

Full Observability Stack

Structured JSON logging, distributed tracing via OpenTelemetry (W3C TraceContext), Prometheus metrics exposition, and /health readiness endpoints out of the box.

Web3 Integration

Native Ethereum JSON-RPC client, ABI encoding/decoding, smart contract bindings generation, wallet and key management, multi-chain support.

Data Layer

Type-safe ORM with compile-time query validation, migration runner, async connection pooling, Redis caching layer, and first-class PostgreSQL support.

Zero-Cost Abstractions

Monomorphized middleware pipeline, stack-allocated request routing, inline HTTP parsing. The runtime overhead is the kernel io_uring interface and nothing else.

Production Ready

Graceful shutdown with drain, hot config reload, TLS via rustls, internationalization (i18n), CLI scaffolding tool, and Docker-ready builds.
