Specialized Agents

Each agent is purpose-built for a specific phase of the development lifecycle, orchestrated by a central coordinator.

Agent Command Description
Orchestrator hbe:orchestrate Central coordinator — dispatches tasks to specialized agents, manages workflow state and dependencies.
Planner hbe:plan Decomposes feature requests into structured task trees with acceptance criteria and priority ordering.
Architect hbe:architect Designs system architecture, module boundaries, API contracts, and data flow diagrams.
Code Reviewer hbe:review Performs deep code review — style, patterns, complexity, maintainability, and best practice compliance.
Security Reviewer hbe:security Scans for vulnerabilities, insecure patterns, dependency risks, and OWASP compliance issues.
TDD Guide hbe:tdd Guides test-driven development workflow — writes failing tests first, then implementation to pass.
Build Error Resolver hbe:fix-build Diagnoses and resolves compilation errors, dependency conflicts, and CI pipeline failures.
E2E Runner hbe:e2e Generates and executes end-to-end tests, validates integration points and user workflows.
Refactor Cleaner hbe:refactor Identifies code smells and applies safe refactoring — renaming, extraction, simplification, deduplication.
Ralph (Autonomous) hbe:ralph Self-supervised autonomous loop — iteratively plans, executes, reviews, and fixes without human intervention.

Core Features

🎯 Agent Orchestration

The Orchestrator agent coordinates all 10 specialized agents in a dependency-aware pipeline. Tasks are dispatched in parallel when possible and sequenced when required, with automatic retry and fallback strategies.

🧪 TDD Workflow

Built-in test-driven development cycle: write failing tests → implement code → verify pass → refactor. The TDD Guide agent enforces red-green-refactor discipline across all supported languages.

🔒 Security Review

Automated security scanning with the Security Reviewer agent. Detects injection vulnerabilities, insecure dependencies, hardcoded secrets, and OWASP Top 10 compliance gaps before code reaches production.

🔁 Closed-Loop Learning

Ralph's autonomous loop applies feedback from each cycle to improve subsequent iterations. Build errors, test failures, and review comments feed back into planning for continuous improvement.

Quick Start

1. Clone the repository git clone https://github.com/ViewWay/hermes-by-everythings.git cd hermes-by-everythings
2. Install for your platform # Claude Code cp -r .claude/ ~/.claude/ # Hermes Agent cp -r .hermes/ ~/.hermes/ # OpenCode / OpenClaw cp -r .opencode/ ~/.opencode/
3. Start coding # Plan a feature hbe:plan "Add user auth" # Run full orchestration hbe:orchestrate "Implement REST API" # Let Ralph handle it hbe:ralph "Fix all failing tests"

Ralph Autonomous Loop

  ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
  │          │     │          │     │          │     │          │
  │  PLAN    │────▶│  EXECUTE │────▶│  REVIEW  │────▶│   FIX    │
  │          │     │          │     │          │     │          │
  └──────────┘     └──────────┘     └──────────┘     └──────────┘
       ▲                                                   │
       │                                                   │
       └───────────────── FEEDBACK ◀───────────────────────┘

  Repeat until:
    ✓ All tests pass
    ✓ No build errors
    ✓ Security review clean
    ✓ Code review approved

Ralph operates as an unattended autonomous agent that iterates through plan → execute → review → fix cycles. Each iteration incorporates feedback from the previous one, creating a closed-loop learning system that converges on high-quality output without human intervention.
